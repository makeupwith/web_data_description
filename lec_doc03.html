本実験の手順を以下に示す。

3.1　Webデータ記述基礎内容

下記の内容でWebサイトを公開することとする。

・W3Cとアクセシビリティおよびバリデーションの関係とその説明

・アクセシビリティおよびバリデーションに利用したツールの説明

・評価実験結果（表にしてまとめること）

・考察

また、レイアウト構成は下記を含めることとする。

・ヘッダー

・トピックパス

・ナビゲーションバー

・フッター（コピーライトを含む）

3.2　アクセシビリティ評価実験
3.2.1　アクセシビリティとは

「アクセシビリティ」とは，「近づきやすさ」「接近容易性」という意味で，社会生活上のあらゆる場で，年齢や身体的制約・利用環境等に関係なく多くの人々が利用しやすいものであること（アクセシブルであること）である．公的なガイドラインには，下記のものがある

・W3C/WAI WCAG 1.0 および 2.0

Web技術の標準化団体であるW3Cが制定したガイドライン．バージョン1.0は1999年5月，バージョン2.0は2006年に制定

・JIS X 8341-3

2004年6月20日に日本で制定されたJISの高齢者・障害者等配慮設計指針

Webドキュメント以外でも，リンク先を別ウィンドウで開かないような工夫も必要である.
これは，視覚に障害のある方や高齢者など多くの利用者が，新しいウィンドウが開いたことに気づかないか，その変化に戸惑う場合があるための対応である．前のページを表示したり，不要なウィンドウを閉じたりする操作は，上肢に障害のある利用者を含む多くの利用者にとって困難になる．
また，必要以上に多くのウィンドウを開くと，サイトを表示している機器に負荷がかかるためコンテンツの表示速度が遅くなることもある．
以上の点から，新規ウィンドウを開くかどうかはユーザの判断と操作に委ねる方が望ましいと考え，アクセシビリティの観点からはリンク先を別ウィンドウで開かない設計が望ましい．



3.2.2　アクセシビリティ・チェックツール

・The W3C Markup Validation Service

インターネット技術の標準化と推進を目的とする国際的な学術団体であるW3C（World Wide Web Consortium）が提供する「W3C MarkUp Validation Service」で，XHTMLの仕様に合致していることを確認できる．

・Web Content Accessibility Guidelines

W3C のWAI が策定したガイドライン“Web Content Accessibility Guidelines”に準拠している．

・W3C CSS 検証サービス

同じくW3Cのサービスで，CSS（Cascading Style Sheetsの略で，W3Cによって規格化された，HTMLで記述された文書にレイアウトスタイルを定義するためのマークアップ言語）の仕様に合致していることを確認できる．



3.2.3　アクセシビリティ評価実験

次の評価実験を行い，考察せよ．

1. アクセシビリティ・チェックツールを用いて，様々なWebページ「10〜20ページ」の調査（チェック）．「個人サイト」と企業や大学といった「団体サイト」の両方のサイトを含めること（第1回目の実験）．

2. 「index.html」を構築するサイトのトップページとして作成し，第1回目の実験終了時までにmoodleにURLを提出（最初の1回目終了時点の内容は「Hello World!」で良い）

3. タイトルバナーの画像の作成（第2回以降の実験）．

4. 調査結果と考察をWebドキュメントとして作成（第2回以降の実験）．

5. 4.で作成したドキュメントごとに１．と同様にチェックし，アイコン画像を取得し，全てのWebドキュメント内に表示せよ（第2回以降の実験）．

3.3　Webサイト構築

Webサイトを構築し，レポートとして提出せよ．Webサイトの構成は3.1で指定した内容とする．また，各Webドキュメントを作成する際には必ずアクセシビリティの確認をし，取得したアイコン画像を全ページ内に表示することとする．
